# Torch K-pop FaceCam


![intro](./document/intro.gif)  

***Torch K-pop FaceCam*** generates videos centered on individual K-pop members from group performance footage. By combining various deep learning modules for multiple object tracking, face recognition, and re-identification, it overcomes challenges like occlusions, nonlinear movements, and camera shifts by identifying and reassigning consistent IDs using facial and body features.

## 1. Introduction

### Problem Statement
Creating an individual member's FaceCam using a standard Multiple Object Tracking (MOT) model alone is highly challenging. K-pop performance videos typically feature severe occlusions from complex dance choreography, nonlinear movements, and rapid camera shifts. 

### Our Approach
![model figure](./document/model_figure.png)  
To solve this problem, we developed a sequential deep learning pipeline that integrates a face recognition model and a person re-identification (Re-ID) model with an MOT model. The initial IDs generated by the tracking model are processed through these modules, which merge and reassign consistent IDs for each actual group member.

### Key Features
* **Tracking:** Utilizes OC-SORT as the baseline tracking model. Its motion-sensitive tracking helps minimize ID-switching.
* **ID-Switch Prevention:** Assigns a new ID when the Intersection over Union (IoU) between bounding boxes is high (indicating severe overlap) to prevent incorrect ID switching.
* **Optimal Sampling:** Prioritizes sampling bounding boxes with low IoU to extract high-quality facial features and identify the K-pop group.
* **Face Recognition:** Identifies people by performing face detection and recognition within the tracking bounding boxes across all frames.
* **Re-identification (Body Features):** When facial information is unavailable (e.g., when a member turns their back), the re-identification model utilizes physical characteristic similarity to maintain tracking.
* **Flexible Video Generation:** Offers options to generate a stabilized, tightly-focused video perfectly centered between the eyes or a more natural-looking video using trajectory smoothing techniques.

## 4. Technical Details
  
### 1) Tracker
Tracking is powered by OC-SORT. We used pre-trained weights from the CrowdHuman dataset, removed hard augmentations, and fine-tuned it using soft augmentations. However, due to the inherent limitations of tracking models in dynamic scenes, it may assign more IDs than the actual number of people and occasionally suffer from ID-switching.
  
### 2) Postprocessor
The Postprocessor refines the tracker's output and serves three primary functions:  
1. **Prevents ID-Switching:** Assigns a new ID when severe bounding box overlap is detected (IoU > threshold).  
2. **Contextual Enrichment:** Adds information regarding bounding box overlaps and depth to help the sampler choose the best frames.  
3. **Noise Filtering:** Deletes track IDs that appear for an extremely short duration, as these are highly likely to be noise.  
  
### 3) Sampler
The Sampler extracts an optimal number of non-overlapping bounding box images per track ID. From these extracted samples, the image yielding the most promising face recognition result is selected to act as the **Body Anchor**. This is because images with good facial recognition results and no overlapping bounding boxes are more likely to contain high-quality full-body characteristics.
  
### 4) Group Recognizer
The Group Recognizer employs a voting mechanism. The Sampler performs face detection and recognition on the extracted images. The system then votes for the group that shows the highest similarity to pre-calculated group embeddings to accurately determine the K-pop group present in the video.  
  
### 5) Face Feature Extractor
This extractor operates on the sampled images from each tracked ID:  
1. Detects the face within the tracked bounding box.  
2. Extracts features from the detected face.  
3. Compares the extracted features against the pre-computed face embeddings of the identified group members to determine the most likely identity.  
  
### 6) Body Feature Extractor
The Body Extractor acts as a supplement:  
1. Creates a "Body Anchor" by averaging body features from samples with high face recognition confidence.  
2. Extracts body features from all tracking bounding boxes and compares them against the body anchor to predict the member's identity across the video.
  
### 7) Predictor
The Predictor assigns a final member identity to each ID by combining face and body prediction similarities. It weighs the results based on confidence:  
* **High Face Confidence:** Relies heavily on the face recognition results.  
* **Medium Face Confidence:** Balances both body and face similarity scores.  
* **Low Face Confidence (e.g., turned away):** Assigns greater weight to the body feature similarity.  
After the prediction process, only one member is assigned to each ID.  
  
### 8) Video Generator
The Video Generator is responsible for calibrating the model's predictions and editing the final output:  
1. **Centering:** Anchors the center of the video frame precisely on the middle of the forehead to create a stable FaceCam effect.  
2. **Dynamic Interpolation:** When the model fails to detect a member, the generator defaults to the entire screen, but interpolates the cutoff moments so the transition changes smoothly.  
3. **Trajectory Smoothing:** Applies smoothing algorithms to the tracking trajectory if the user opts for a more natural viewing experience.

### 9) MLOps Pipeline (Online Training)
Because performers' outfits and appearances change drastically between videos, creating static feature anchors beforehand is difficult. To address this, we implemented an MLOps pipeline using Apache Airflow. The pipeline captures bounding box images during inference and sends them in batches to the model trainer daily, utilizing contrastive learning to continuously improve the body feature extraction model's performance without requiring manual labeling.
